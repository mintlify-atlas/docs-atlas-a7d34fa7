---
title: "Configuration"
description: "All environment variables for the Heimdall control plane and node agents, with defaults and production guidance."
---

Heimdall is configured entirely through environment variables. Load them from a `.env` file in the project root or export them in your shell before running `start.sh` or `start-agent.sh`.

<Warning>
  `INFRA_API_KEY` must be set to the **same value** on the control plane and every node agent. A mismatch causes all webhook deliveries to fail with a signature verification error.
</Warning>

## Authentication

<ParamField path="INFRA_API_KEY" type="string" required>
  HMAC secret shared between the control plane and all agents. Used to sign and verify every webhook payload. Must be set to the same value on all nodes. There is no default — the application exits at startup if this is unset in any environment other than `dev`/`test` with `HEIMDALL_ALLOW_DEFAULTS=1`.
</ParamField>

## Network

<ParamField path="INFRA_API_URL" type="string" default="http://localhost:8000">
  Base URL of the control plane. Node agents read this to construct the webhook delivery URL (`INFRA_API_URL/webhook`). Set to the publicly accessible address of your control plane host in multi-node deployments.
</ParamField>

## Runtime environment

<ParamField path="HEIMDALL_ENV" type="string" default="dev">
  Runtime environment name. Set to `prod` for production deployments. When set to `prod`, any missing required secrets cause the application to exit immediately at startup rather than falling back to dev defaults.
</ParamField>

<ParamField path="HEIMDALL_ALLOW_DEFAULTS" type="string" default="0">
  Set to `1` to allow development defaults for secrets in `dev` or `test` environments. When enabled, missing secrets print a warning instead of raising an error. **Never enable this in production.**
</ParamField>

<ParamField path="HEIMDALL_AGENT_DISABLE_BG" type="string" default="0">
  Set to `1` to disable background tasks on the node agent. Useful for debugging or running the agent in a minimal mode where only the HTTP server starts.
</ParamField>

## Health monitoring

<ParamField path="FAIL_THRESHOLD" type="number" default="3">
  Number of consecutive failed heartbeats before a node is marked `OFFLINE`. Increase this value to tolerate intermittent network issues; decrease it for faster failure detection.
</ParamField>

<ParamField path="MONITOR_INTERVAL_SECONDS" type="number" default="5">
  How often (in seconds) the control plane polls node heartbeats. Lower values give faster health detection at the cost of higher network and CPU overhead.
</ParamField>

<ParamField path="HEARTBEAT_TIMEOUT_SECONDS" type="number" default="2">
  Timeout in seconds for individual heartbeat HTTP requests from the control plane to each agent. Requests that exceed this timeout count toward the node's fail threshold.
</ParamField>

## Security

<ParamField path="WEBHOOK_TTL_SECONDS" type="number" default="60">
  Maximum age (in seconds) of a webhook timestamp before the control plane rejects the payload. Protects against replay attacks. Clocks on the control plane and all agents must be synchronized within this window.
</ParamField>

## Integrations

<ParamField path="DISCORD_TOKEN" type="string" required>
  Bot token for the Discord integration. Required only if you are running the Discord bot (`discord_bot/bot.py`). Obtain this from the [Discord Developer Portal](https://discord.com/developers/applications).
</ParamField>

## Example .env file

```bash .env
INFRA_API_KEY=change-me
INFRA_API_URL=http://localhost:8000
HEIMDALL_ENV=dev
HEIMDALL_ALLOW_DEFAULTS=1
HEIMDALL_AGENT_DISABLE_BG=0
FAIL_THRESHOLD=3
WEBHOOK_TTL_SECONDS=60
MONITOR_INTERVAL_SECONDS=5
HEARTBEAT_TIMEOUT_SECONDS=2
```

## Production checklist

Before deploying to production, verify the following:

<Check>
  `INFRA_API_KEY` is set to a strong, randomly generated secret (not `change-me`).
</Check>
<Check>
  `INFRA_API_KEY` is identical on the control plane and every node agent.
</Check>
<Check>
  `HEIMDALL_ENV` is set to `prod`.
</Check>
<Check>
  `HEIMDALL_ALLOW_DEFAULTS` is set to `0` or unset.
</Check>
<Check>
  `INFRA_API_URL` points to the publicly accessible address of your control plane, not `localhost`.
</Check>
<Check>
  Clocks on all hosts are synchronized (NTP) so webhook timestamp validation works within `WEBHOOK_TTL_SECONDS`.
</Check>
<Check>
  `DISCORD_TOKEN` is set if you are running the Discord bot.
</Check>
