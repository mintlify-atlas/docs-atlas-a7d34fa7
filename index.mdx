Heimdall is a distributed infrastructure orchestration platform. It gives you a central control plane to deploy Nix-flake-defined services across multiple hosts, track their health in real time, and operate them from Discord — without touching a terminal on every machine.

## How it works

Heimdall has two components that work together:

**Control Plane** — A FastAPI server that stores node and service state in SQLite, accepts signed webhook events from agents, and exposes a REST API for deployments, rollbacks, and auditing.

**Node Agent** — A lightweight FastAPI process that runs on each host. It receives deployment commands from the control plane, executes `nix run` to start services, streams logs back, and sends health status updates via signed webhooks.

<Steps>
  <Step title="Register your nodes">
    Add each host to Heimdall so the control plane knows where to send deployments.
  </Step>
  <Step title="Declare your services">
    Point Heimdall at a Nix flake that defines how your service is built and run.
  </Step>
  <Step title="Deploy">
    Trigger a deployment via the REST API or a Discord slash command — Heimdall handles the rest.
  </Step>
  <Step title="Monitor">
    Watch real-time health status, stream logs, and roll back instantly if something goes wrong.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="HMAC-signed webhooks" icon="shield-halved" href="/architecture/webhooks-and-security">
    All agent-to-control-plane communication is authenticated with HMAC-SHA256 signatures and replay-attack protection via timestamps.
  </Card>
  <Card title="Nix flake manifests" icon="snowflake" href="/guides/nix-flake-manifest">
    Services are defined entirely in Nix flakes, giving you reproducible builds and a single source of truth for deploy, teardown, and healthcheck commands.
  </Card>
  <Card title="Operation audit log" icon="clock-rotate-left" href="/operations/deploy">
    Every deployment, rollback, and teardown is recorded with who triggered it, when it ran, and whether it succeeded.
  </Card>
  <Card title="Multi-node support" icon="server" href="/guides/multi-node-setup">
    Register any number of nodes and deploy different services to different hosts — or bulk-deploy everything at once.
  </Card>
