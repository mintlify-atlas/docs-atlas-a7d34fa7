## Prerequisites

Before you begin, make sure you have the following installed:

- **Python virtual environment** — `python -m venv .venv` and activate it
- **[Nix](https://nixos.org/)** with flakes enabled (`experimental-features = nix-command flakes` in your Nix config)
- **tmux** — required by the launch scripts to manage processes
- **git**

<Steps>

<Step title="Clone the repository">

```bash
git clone https://github.com/Blazzzeee/heimdall_py.git
cd heimdall_py
```

</Step>

<Step title="Enter the dev shell">

The Nix flake provides a fully reproducible shell with all system-level dependencies pinned. Choose whichever method fits your workflow:

<CodeGroup>

```bash nix develop
nix develop
```

```bash direnv allow
direnv allow
```

</CodeGroup>

<Tip>
  If you use direnv, the shell activates automatically every time you `cd` into
  the project directory. Run `direnv allow` once after cloning.
</Tip>

</Step>

<Step title="Install Python dependencies">

```bash
pip install -r requirements.txt
```

This installs FastAPI, Uvicorn, SQLAlchemy, httpx, discord.py, and all other runtime and test dependencies.

</Step>

<Step title="Configure environment variables">

Copy the example file and fill in your values:

```bash
cp .env.example .env
```

```ini .env.example
INFRA_API_KEY=change-me
INFRA_API_URL=http://localhost:8000
HEIMDALL_ENV=dev
HEIMDALL_ALLOW_DEFAULTS=1
HEIMDALL_AGENT_DISABLE_BG=0
FAIL_THRESHOLD=3
WEBHOOK_TTL_SECONDS=60
MONITOR_INTERVAL_SECONDS=5
HEARTBEAT_TIMEOUT_SECONDS=2
```

| Variable                    | Description                                                                                                               |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `INFRA_API_KEY`             | Shared secret used to authenticate API requests and sign HMAC webhooks between agents and the control plane.             |
| `INFRA_API_URL`             | Base URL of the control plane. Agents append `/webhook` to this to construct the event delivery URL.                      |
| `HEIMDALL_ENV`              | Runtime environment. `dev` enables safe defaults; set to `prod` to require all secrets explicitly.                        |
| `HEIMDALL_ALLOW_DEFAULTS`   | When `1`, missing secrets in `dev`/`test` fall back to built-in defaults with a warning instead of raising an error.      |
| `HEIMDALL_AGENT_DISABLE_BG` | Set to `1` to disable the agent's background health-check and log-flush tasks. Useful for isolated testing.               |
| `FAIL_THRESHOLD`            | Number of consecutive failed heartbeat polls before a node is marked `OFFLINE`.                                           |
| `WEBHOOK_TTL_SECONDS`       | Maximum age (in seconds) of a signed webhook timestamp. Requests outside this window are rejected as replays.             |
| `MONITOR_INTERVAL_SECONDS`  | How often (in seconds) the control plane polls node heartbeats.                                                           |
| `HEARTBEAT_TIMEOUT_SECONDS` | HTTP timeout for each heartbeat request from the control plane to an agent node.                                          |

<Warning>
  `INFRA_API_KEY` is required by both `start.sh` and `start-agent.sh`. The
  scripts will exit immediately if it is not set. Do not use the placeholder
  value `change-me` in any environment exposed to a network.
</Warning>

</Step>

<Step title="Start the control plane">

```bash
chmod +x ./start.sh && ./start.sh && tmux attach -t heimdall
```

The script will:

1. Kill any processes already bound to ports `8000`, `8001`, and `5000`
2. Start a tmux session named `heimdall` with four windows: `control`, `agent`, `bot`, and `logs`
3. Launch the FastAPI control plane on port `8000`
4. Bootstrap a `local-agent` node in the database
5. Register two example services (`service-1`, `worker-1`) unless `PROD=true`
6. Start the node agent on port `8001`
7. Start the Discord bot

<Note>
  The session exposes the control plane at `http://localhost:8000` and the local
  agent at `http://localhost:8001`. Use `tmux kill-session -t heimdall` to stop
  everything.
</Note>

</Step>

<Step title="Start a node agent on another machine (optional)">

When operating a multi-machine setup, run only the agent on each worker node. Point it back at the control plane running on the primary host.

```bash
chmod +x ./start-agent.sh && ./start-agent.sh && tmux attach -t heimdall-agent
```

Make sure `INFRA_API_URL` and `INFRA_API_KEY` in `.env` on the worker node match the control plane's address and shared secret.

The agent starts on port `8001` by default. Override with `HEIMDALL_AGENT_PORT` if that port is taken.

</Step>

<Step title="Verify with a health check">

Once the control plane is up, confirm it is responding:

```bash
curl http://localhost:8000/health
```

Expected response:

```json
{"status": "ok", "timestamp": 1712345678.123}
```

<Note>
  If `curl` returns a connection error, check that the control plane window
  inside tmux started without errors: `tmux attach -t heimdall` and switch to
  the `control` window with `Ctrl-b 0`.
</Note>

</Step>

</Steps>
